git command reference

git init
initialize an empty repo in current directory 
git config

examples:
git config –global user.name “katze”
git config –global user.email [myemailaddress]@[myemailprovider].com

git commit -m [comment] [filename]

commit change

git clone

example: git clone https://[userid]@[someurl]/[projectname].git

clone a repository to local directory

.gitignore

define files to exclude and include

example:

$ cat .gitignore
*.a
!lib.a

exclude *a except lib.a

git status

find files that are not in sync

rm [filename]
git rm [filename]

remove a file from file system and git

git rm --cached [filename]

remove a file from git but not file system

git stash

save the working directory but do not commit (not yet sure what this means)

git mv [old file] [new file]

rename/restructure a file on both file system and git

git log

display commit logs

git tag [tagname]

tag the current commit

git push

push things into remote repo

git commit --amend

amend most recent commit before it’s pushed into remote repo (i dont really like this..)

git commit --date="[new date in `date` output format]" -m "add date"

commit with different date (i dont really like this either..)

git reset [filename]

undo changes not yet committed

git reset --soft HEAD^

undo last commit but keep the index (not yet sure what keep the index means..)

git checkout [filename]

check out a file from last commit and discard uncommitted changes

git remote -v

identify remote repo

git pull

pull changes from origin repo

git remote add [repo name] [repo url]

add a remote repo

git rebase
git push

rebase and push commit to remote

git diff [filename]
find modification to file since last commit 
git blame [filename]
find out who edited which line 
git branch [branch name]
create new branch, to work on new features that has potential to break things independently 
git checkout -b [branch name]
create a new branch and switch to it 
git checkout [tag name]
git checkout tags/[tag name]

check out files from a tag
use the 2nd command if same name is both a branch name and a tag name

git branch [new branch name] -v [previous commit id from `git log` output]
create a branch from a previous commit 
git branch -d [branch name]
delete a branch 
git push [remote repo name] [branch name]
push a branch to a remote repo remote repo name can be found in `git remote -v` output, likely to be 'origin' 
git merge [branch name]
merge another branch into current branch 
git fetch
fetch new change from remote repo without merging with local repo 
git rebase master [branch name]
rebase a branch onto master branch 
git gc
git repack
optimize how the repo is packed, cleanup redundant packages (not yet understand the benefit of this 
git checkout [another branch]
git log --oneline
git checkout [current branch]
git cherry-pick [commit id]
commit a commit from another branch and merge into current branch 
git grep [keyword]

works line normal grep
grep in current repo

git rebase -i HEAD~[number of commit from most recent one, in `git log --oneline` output]

example: git rebase -i HEAD~2

use case: correct typo in commit log
in vi editor, replace ‘pick’ with ‘reword’
save and quit
will get a chance to correct the wording in 2nd vi editor
save and quit

git rebase -i HEAD~[number of commit from most recent one, in `git log --oneline` output]

use case: merge multiple commits
in vi editor, replace ‘pick’ with ‘squarsh’
save and quit
will get a chance to rephrase commit log in 2nd vi editor
save and quit

git merge --squash [branch name]
git commit

merge all commits of [branch name] as a single commit

git rebase -i HEAD~[number of commit from most recent one, in `git log --oneline` output]

example: git rebase -i HEAD~2

use case: order previous commits

git bisect start
git bisect bad [one example bad commit; latest one possibly]
git bisect good [one example good commit]
git bisect run [command than returns exit code 0 if good and other code if bad]

after running above, git will output the first bad commit and revert the directory to first good commit
use case: to find a bug introduce along the way

git add -p
This opens a vi editor, that contains output of `git diff`, delete the change not yet feature for staging to commit use case: changes are made for multiple features, only stage the commit for certain feature 
git add
git add -A stages All git add . stages new and modified, without deleted git add -u stages modified and deleted, without new 
git reflog
display git command history 
git revert [commit id]
revert a bad commit that has been pushed use `git reflog` or `git log --oneline` to find commit id 
git reflog
git checkout [deleted commit id]

use case: restore latest commit which was deleted via git reset --hard HEAD^

git branch
git merge [branch name]
vi [file with conflict]
git add [file with resolved conflict]
git commit

merging branches and resolving conflicts

git submodule add [sub module repo url] [target directory]

add a sub module to be managed independently

Concepts

pull = fetch + merge

merge will keep different streams (pros: non-destructive; useful for coorperation among big team)
rebse will merge everything into 1 sequential stream (pros: cleaner project history; uesful for independent developer)

tag: a version of a particular branch at a moment of time
branch: a thread of development
