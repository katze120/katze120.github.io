2015-08-20

openstack instances are up!

My OpenStack instances are up on network!!

First I had to cleanup all networks, subnets, and routers that were installed by default. Somehow they did not match the requirement, eg. public network did not have the correct IP range, provider info was not correct and not easily editable, etc.
I followed steps on https://saliux.wordpress.com/2014/04/16/clean-up-openstack-router-and-networks to cleanup.
And then steps on https://www.rdoproject.org/Neutron_with_existing_external_network to recreate public and private network, with public network being existing network on the hypervisor aka openstack server
Then https://www.rdoproject.org/Neutron_with_existing_external_network to launch the instance. Needed to add additional security rules for ssh and ping; instances need to be created with private IP; then assocaite a floating IP in public subnet to it.
Lesson learnt: follow instructions!
Next step: try to understand each step I did; now I can also start reading OpenStack manuals to take a closer look at each component.
Look forward to more fun!

Use cases

Configure a static IP

  1. Input the configuration 

    # vi /etc/sysconfig/network-scripts/ifcfg-enX
    or 
    # nmtui edit enX

  2. Restart the network interface

    # nmcli d disconnect enX
# nmcli d connect enX
    or 
    # nmcli c down enX
# nmcli c up enX
    or 
    # systemctl restart NetworkManager

  3. Verify

    # ip a

Configure a second IP on the existing network connection profile

# nmcli con mode enX +ipv4.addresses "xx.xx.xx.xx/xx"
# nmcli con up xnX

RHEL 7 xfs

XFS is now the default file system. XFS was already fully supported in RHEL6 but was not the default.

Reference Table for ext4 and XFS Commands

Task ext4 xfs
Creating a file system mkfs.ext4 mkfs.xfs
Mounting a file system mount mount
Resizing a file system resize2fs xfs_growfs (increase only)
Repairing a file system e2fsck xfs_repair
Changing the label on a file system e2label xfs_admin -L

Command Examples

Create a new LV


 # lvcreate -n “lv_test” -L 1g vg_rhel


Create a new XFS file system


 # mkfs.xfs /dev/vg_rhel/lv_test


Increase the LV size


 # lvextend -L +2G /dev/vg_rhel/lv_test


Increase the file system size


xfs_growfs [filesystempath]


Notes:
This is only possible for a mounted file system Reducing file system size is not possible even in unmounted status.

Repair a file system


# unmount /dev/vg_rhel/lv_test
# xfs_repair /dev/vg_rhel/lv_test


Assign a label


 # umount /dev/vg_rhel/lv_test
# xfs_admin -L “Label” /dev/vg_rhel/lv_test


Read the file system label

# xfs_admin -l /dev/vg_rhel/lv_test
2015-08-08

RHEL 7 systemd

Reference: https://access.redhat.com/articles/754933

Overview of systemd

  • New service manager for Linux
  • Replaces System V init and Upstart
  • Backward compatability for init scripts
  • Benefits:
  1. Fast boot and shutdown times because of parallelization of boot services and dependency tracking
  2. Autospawn and respawn of services
  3. Tight integration with cgroups
  4. Logs early kernel boot up messages

Basics

The global systemd configuration is stored in the /etc/systemd directory.

The service configuration files are located in the /usr/lib/systemd/system directory

Custom service configuration files are stored in the /etc/systemd/system directory

Get the current release of systemd

$ systemctl --version
systemd 208
+PAM +LIBWRAP +AUDIT +SELINUX +IMA +SYSVINIT +LIBCRYPTSETUP +GCRYPT +ACL +XZ

Service Management

Some service and chkconfig commands continue to work, and will be translated to systemd equivalents.

# chkconfig httpd
Note: Forwarding request to 'systemctl is-enabled httpd.service'.
enabled

# service httpd start
Redirecting to /bin/systemctl start  httpd.service

Active/Deactivate a particular service at boot



# systemctl enable httpd
ln -s ‘/usr/lib/systemd/system/httpd.service’ ‘/etc/systemd/system/multi-user.target.wants/httpd.service’

# systemctl disable httpd
rm ‘/etc/systemd/system/multi-user.target.wants/httpd.service’


Notes:

In recent versions of systemctl, by default .service suffix will be added If we specify a path, the .mount suffix will be added
* If we specify a device, the .device suffix will be added

Mask/Unmask a service

[root@katzerhel7 ~]# systemctl mask httpd
ln -s '/dev/null' '/etc/systemd/system/httpd.service'
[root@katzerhel7 ~]# systemctl unmask httpd
rm '/etc/systemd/system/httpd.service'

Notes:

  • We can mask any service unit to prevent it from being started manually or by another service
  • We cannot enable a masked service, but we can mask an enabled service

Check whether a particular service is activated at boot time

# systemctl is-enabled httpd
enabled

Notes:

  • Possible outputs are: enabled, disabled, masked

Start, stop, restart, reload a particular service

# systemctl start httpd
# systemctl stop httpd
# systemctl restart httpd
# systemctl reload httpd

Notes:

  • Similar to “service” in RHEL 6 & 5, the reload option is not available for all the services

Check whether a particular service is running

# systemctl is-active httpd
active

Targets/Run levels


This is how systemd targets alight with traditional runlevels:


Traditional runlevel      New target name     Symbolically linked to...
Runlevel 0           |    runlevel0.target -> poweroff.target
Runlevel 1           |    runlevel1.target -> rescue.target
Runlevel 2           |    runlevel2.target -> multi-user.target
Runlevel 3           |    runlevel3.target -> multi-user.target
Runlevel 4           |    runlevel4.target -> multi-user.target
Runlevel 5           |    runlevel5.target -> graphical.target
Runlevel 6           |    runlevel6.target -> reboot.target

Journal analysis

Get the content of the systemd journal

$ journalctl

Notes:

  • To make journalctl work for non-root user, /run/log/journal/[xxxx]/system.journal needs to be readable by the user. Check /usr/lib/tmpfiles.d/systemd.conf too
  • By default, journald logs are stored in the /run/log/journal directory and disappear after a reboot

Get all the events related to particular process in the journal

$ journalctl /sbin/crond
$ journalctl -u crond

View kernal messages from current boot

 $ journalctl -k

Follow journal messages

$ journalctl -f

Get all the events with a particular syslog priority

$ journalctl -p err

RHEL 7 differences with 6

These were notes from late year. Just keeping an record.

Differences between RHEL 7 & 6

Basics

Features/Command/Config file RHEL 7 RHEL 6
Default file system XFS EXT4
Kernel version 3.10.x-x 2.6.x-x
Kernel Code Name Maipo Santiago
Host Name Change /etc/hostname /etc/sysconfig/network

systemd

Services

Features/Command/Config file RHEL 7 RHEL 6
First Process systemd (PID 1) init (PID 1)
List all services systemctl list-unit-files –type=service, ls /{usr/lib,etc}/systemd/system/*.service chkconfig –list, ls /etc/init.d/
List running services systemctl -t service –state=active service –status=all
Start/stop a service systemctl start [name], systemctl stop [name] service [name] start, service [name] stop
Enable/Disable a service systemctl enable [name], systemctl disable [name] chkconfig [name] on, chkconfig [name] off
Check if service is enabled systemctl is-enabled [name] chkconfig [name]
View service status systemctl status [name] service [name] status
Create service file or change config systemctl daemon-reload chkconfig –add [name]

Runlevel

Features/Command/Config file RHEL 7 RHEL 6
View runlevel/target systemctl get-default runlevel
Change runlevel/target systemctl isolate [name].target, init [runlevel] init [runlevel]
default Runlevel /etc/systemd/system/default.target (this is by default linked to the multi-user target) /etc/inittab

Logging

Features/Command/Config file RHEL 7 RHEL 6
Configure logging /etc/rsyslog.conf, /etc/rsyslog.d/*.conf, /etc/systemd/journald.conf /etc/rsyslog.conf
View logs /var/log/, journalctl /var/log/

File system

Features/Command/Config file RHEL 7 RHEL 6
Default File System XFS EXT4
File System Check xfs_repair (XFS does not run a file system check at boot time) e2fsck (File system check would get executed at boot time)
Format disk partition mkfs.filesystem_type (ext4, xfs) mkfs.filesystem_type (ext4, xfs)
Enlarge file system xfs_growfs (xfs_growfs takes mount point as arguments) resize2fs (resize2fs takes logical volume name as arguments)
Shrink file system (XFS cannot currently be shrunk; copy desired data to a smaller file system) resize2fs

Networking

Features/Command/Config file RHEL 7 RHEL 6
Network Bonding Teaming, /etc/sysconfig/network-scripts/ifcfg-team0, DEVICE=”team0”, DEVICETYPE=”Team” Bonding, /etc/sysconfig/network-scripts/ifcfg-bond0, DEVICE=”bond0”
Network Startup NetworkManager service Network initscripts
default Network device naming en* eth*

Clustering

Features/Command/Config file RHEL 7 RHEL 6
Cluster Resource Manager Pacemaker Pacemaker (6.5, 6.6) / Rgmanager
